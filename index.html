Why These Tools Are Needed for Verifying LLM Answers

Large Language Models (LLMs) can generate fluent but sometimes hallucinated or logically inconsistent answers. Each tool in this project addresses a different failure mode and contributes to making LLM outputs trustworthy.

1. Discourse Parser

Checks the form of reasoning by extracting rhetorical relations such as evidence ‚Üí claim, cause ‚Üí effect, or contrast. It ensures the explanation flow is coherent.

1') FastAPI to Discourse Parser

Exposes the parser as a service endpoint so other tools can automatically call discourse verification inside a pipeline.

2. Logic Program Runner

Checks the content of an LLM answer against a formal ontology. If the claims do not unify with the rules (e.g., medical knowledge base), they are flagged as inconsistent.

3. Rule Attenuation Engine

Introduces flexibility by relaxing strict rules step by step. This allows us to test whether a diagnosis is still plausible when some conditions are missing or weakened.

4. Diagnosis-Making Verifier

Integrates discourse parsing and logic checks into a final truth meter for LLM answers. If discourse and ontology align ‚Üí ‚úÖ verified. Otherwise ‚Üí ‚ùå unreliable.
